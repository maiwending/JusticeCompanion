# Justice Companion

An AI-assisted tool that helps incarcerated individuals get timely, informed responses to their messages. When a detained person sends a message through [SmartInmate](https://www.smartinmate.com), Justice Companion detects it, generates a draft reply using a local AI model, and delivers it — so their family members and support network can respond quickly and thoughtfully even when life gets busy.

> **The AI never communicates directly with the detained person.** Every reply is reviewed and approved by a human before it is sent.

---

## How It Works (Stage 1)

1. The bot logs into SmartInmate on a schedule (every 10 minutes).
2. It scans the inbox for unread messages.
3. For each message, it generates a draft reply using a locally-running LLM via [LM Studio](https://lmstudio.ai).
4. The draft is automatically sent back through SmartInmate, prefixed with a clear AI disclaimer.
5. All conversations are logged locally for review.

```
SmartInmate inbox
      │ (Selenium, every 10 min)
      ▼
main.py — reads unread messages
      │
      ▼
ai_chat.py — generates reply via LM Studio (local LLM)
      │
      ▼
SmartInmate — reply posted back
```

## Prerequisites

- Python 3.9+
- Google Chrome + [ChromeDriver](https://chromedriver.chromium.org/) matching your Chrome version
- [LM Studio](https://lmstudio.ai) running locally with a model loaded (default endpoint: `http://localhost:1234`)
- A [SmartInmate](https://www.smartinmate.com) account linked to the incarcerated person
- A Google Cloud project with the Gmail API enabled (for email-alert watching)

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/your-username/JusticeCompanion.git
cd JusticeCompanion
```

### 2. Create a virtual environment and install dependencies

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Configure credentials

Copy the example environment file and fill in your values:

```bash
cp .env.example .env
```

Edit `.env`:

```env
GMAIL_USER=your_gmail_address@gmail.com
SMARTINMATE_USERNAME=your_smartinmate_username
SMARTINMATE_PASSWORD=your_smartinmate_password
```

### 4. Set up Google OAuth (for Gmail watching)

1. Go to [Google Cloud Console](https://console.cloud.google.com) and create a project.
2. Enable the **Gmail API**.
3. Create OAuth 2.0 credentials (Web application type).
4. Download the JSON and save it as `credentials.json` in the project root.
   - Use `credentials.json.example` as a reference for the expected format.
5. On first run, a browser window will open asking you to authorize Gmail access. A `token.json` file will be generated automatically and reused on future runs.

### 5. Load a model in LM Studio

Open LM Studio, download a model (e.g., `meta-llama-3-8b-instruct`), and start the local server. The default API endpoint is `http://localhost:1234/v1/chat/completions`. Update `ai_chat.py` if your endpoint or model name differs.

### 6. Run the bot

```bash
python main.py
```

The bot will check for unread messages every 10 minutes. Press `Ctrl+C` to stop.

## Security Notes

- **Never commit `.env`, `credentials.json`, or `token.json`** — all three are listed in `.gitignore`.
- Credentials are loaded exclusively from environment variables via `python-dotenv`.
- The Gmail integration uses read-only scope (`gmail.readonly`).

## Roadmap

Stage 1 (current) is a working prototype that automatically generates and sends AI replies. **Stage 2** will shift the workflow to a human-in-the-loop mobile experience:

- **Push notifications** — when an inmate sends a message, the family member receives an instant notification on their phone (iOS & Android).
- **Draft review screen** — the notification opens a mobile app showing the inmate's message alongside the AI-drafted reply.
- **Human approval required** — the family member can Send, Edit, or Decline the draft. Nothing is ever sent without their explicit action.
- **Backend**: Firebase (Firestore + Cloud Messaging) to sync drafts and decisions in real time.
- **Mobile app**: React Native / Expo for a cross-platform experience.

See [`plan.md`](./plan.md) for the full Stage 2 architecture and implementation plan.
